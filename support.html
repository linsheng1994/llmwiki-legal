产品概览

LLM Wiki App 是一个本地优先的 macOS 知识库工作台，用于把来源文件、网页、PDF、图片 OCR、知识页、审阅记录、查询结果和校验报告组织到同一个工作区中。

本地工作区 围绕你主动选择的本地文件夹运行，而不是把全部内容托管到云端。
Inbox 工作流 新材料先进入 Inbox，再整理成来源笔记、知识更新和待审 patch。
可选模型接入 支持连接兼容 OpenAI Chat Completions 的远程或本地模型接口。
可审计 任务、审阅和写入轨迹都保留在工作区内，方便回看与回滚。

快速开始

  1. 打开应用，选择一个本地文件夹作为工作区。
  2. 在“工作区”页初始化结构，生成 Inbox、wiki、review、tasks 等目录。
  3. 进入“来源”导入文件或链接。
  4. 点击“开始整理”，把 Inbox 材料转成来源笔记、知识提案或待审 patch。
  5. 进入“审阅”“知识库”“查询”“校验”继续维护工作区。

常见问题

应用是否必须联网？

不必须。你可以只使用本地工作区、OCR、浏览和部分本地规则流程。只有你主动导入网页、发现远程来源或配置并调用模型时，应用才会联网。

为什么某些功能提示需要先配置模型？

某些 workspace schema 会要求 durable 知识维护必须由模型生成。如果当前工作区处于严格 durable 模式，而你还没有完成模型接入，应用会阻止相关写入动作。

API Key 存在哪里？

API Key 保存在 macOS Keychain 中，不会明文写入工作区文件。

watched folder 是什么？

watched folder 是你为某个工作区授权的本地目录。应用可以监测该目录中的新文件并自动把它们收进工作区 Inbox。

如果模型服务是我自己填的地址，数据由谁负责？

如果你把内容发送到自定义 Base URL，对应服务的运营方将按其自身政策处理数据。请在使用前确认你信任该服务并理解其保留与审计策略。

隐私说明

LLM Wiki App 默认把工作区文件、索引、审阅记录和任务历史保存在你的 Mac 上。当前版本的“匿名使用统计”仅记录本地计数，不会上传到开发者自有外部分析服务。

只有当你主动执行需要联网的动作时，应用才会向目标网站、搜索服务或你配置的模型端点发送必要内容。详情请阅读 完整隐私政策。
